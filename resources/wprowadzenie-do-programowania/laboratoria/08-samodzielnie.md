Tym razem nie jakaś lista zadań, rodem z podręcznika. Próba stworzenia czegoś o pewnej określonej funkcjonalności

Twoim zadaniem jest stworzenie prostej aplikacji do zarządzania listą zadań. Program powinien umożliwiać użytkownikowi dodawanie zadań do listy, wyświetlanie zadań, usuwanie zadań oraz sprawdzanie ich statusu.

Oto wytyczne do zadania:

1. Użytkownikowi powinno być wyświetlone menu z opcjami:
   - Dodaj zadanie
   - Wyświetl zadania
   - Usuń zadanie
   - Zakończ program
2. Program powinien przechowywać listę zadań. Może to być lista Pythona.
3. Po wyborze opcji "Dodaj zadanie," użytkownik powinien móc wpisać treść nowego zadania, a zadanie zostanie dodane do listy.
4. Po wyborze opcji "Wyświetl zadania," program powinien wyświetlić listę zadań, numerując je. Użytkownik powinien widzieć, czy zadanie zostało zakończone (np. "\[x\] Zadanie numer 1" oznacza zakończone zadanie).
5. Po wyborze opcji "Usuń zadanie," użytkownik powinien móc wprowadzić numer zadania, które chce usunąć z listy. Program powinien potwierdzić usunięcie zadania.
6. Po wyborze opcji "Zakończ program," program powinien zakończyć działanie.
7. Program powinien obsługiwać błędy, na przykład jeśli użytkownik spróbuje usunąć nieistniejące zadanie.

Lub inaczej:

Twoim zadaniem jest stworzenie prostej aplikacji restauracyjnej, która umożliwia użytkownikowi zamawianie potraw z menu i wyliczanie łącznej sumy zamówienia. Menu i ceny potraw będą przechowywane w listach.

Oto wytyczne do zadania:

1. Utwórz dwie listy: jedną z menu potraw i drugą z odpowiadającymi im cenami. Każdy indeks na liście menu powinien odpowiadać tej samej pozycji na liście cen.
2. Wyświetl menu użytkownikowi, numerując pozycje na liście.
3. Użytkownik powinien móc wybrać potrawy, które chce zamówić. Użyj pętli do zbierania zamówień użytkownika, aż użytkownik zakończy zamawianie.
4. Po zakończeniu zamawiania, program powinien obliczyć łączną sumę zamówienia na podstawie wybranych potraw i cen.
5. Wydrukuj rachunek, podając wybrane potrawy, ich ceny i łączną kwotę do zapłaty.
6. Program powinien obsługiwać sytuacje, w których użytkownik poda numer potrawy, która nie istnieje w menu.
7. Po zakończeniu zamawiania użytkownik powinien mieć możliwość zakończenia program
